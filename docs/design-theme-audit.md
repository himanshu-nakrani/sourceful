# Design Theme Audit (April 27, 2026)

## Executive summary

The application now follows a **Precision Intelligence Workspace** direction:

- product-led, not marketing-led
- source provenance as the signature visual system
- restrained dark material surfaces
- solid primary actions with accent color reserved for trust, status, and citation cues
- reduced reliance on glow, gradients, ambient background effects, and generic AI language

The strongest design asset is the core workflow: upload documents, ask questions, inspect citations, review retrieval confidence, and save or rerun answers. The visual system should keep making that workflow feel accountable and premium.

## What changed

- The marketing page now opens with a believable workspace preview instead of abstract AI SaaS composition.
- Generic hero language, animated ambient blobs, decorative grid overlays, bento cards, and emoji-like tech visuals were removed.
- The welcome/setup screen now feels like a secure workspace console.
- Shared CSS naming now emphasizes precision/provenance instead of glassmorphism.
- Glow and soft-gradient intensity were reduced across the token system.
- Metadata now uses traceable document intelligence language.

## Design principles

- **Evidence is the premium layer.** Citations, retrieval scores, page anchors, confidence rails, and source state should be visually clearer than decorative effects.
- **Futuristic means precise.** Favor sharp hierarchy, quiet density, strong spacing, and deliberate state changes over neon or visual spectacle.
- **Accent is earned.** Teal, violet, and gradient treatments should be rare and tied to provenance, focus, or primary action.
- **Core app stays useful.** Sidebar density, chat readability, source cards, reduced motion, high contrast, and accent packs remain important.

## Current strengths

- Mature dark-first token system with light mode, high contrast mode, reduced motion support, and accent packs.
- Good separation between navigation, chat, sources, settings, analytics, and workspace management.
- Confidence rails and source cards create a distinctive trust-centered product language.
- Onboarding and marketing now align better with the actual product promise.

## Watchouts

- Avoid reintroducing Sparkles/Zap-style iconography for model or chat features.
- Avoid new background blobs, mesh gradients, aurora animations, or decorative grids.
- Avoid using gradient text for headlines. Keep it for small status/provenance moments only.
- Avoid generic copy like "powered by AI" or "agentic capabilities" unless it is tied to a concrete workflow.
- Keep card radii and shadows restrained, especially in dense app surfaces.

## Recommended next refinements

- Continue replacing any remaining decorative glow with borders, rails, and semantic state.
- Make citation hover previews richer and more keyboard accessible.
- Add compact provenance summaries to long answers: source count, score range, and weakest evidence.
- Consider a polished research layout default for multi-source workflows.
- Run visual checks whenever marketing, onboarding, settings, or chat-empty states change.
